# Dame — 前端重写计划

> 更新：2026-06-15
>
> **dame 不是独立项目。** dame 是对 LingServer Dashboard 主线的**纯前端重写实验**，目标是将其视觉设计、交互动效、布局架构合并回主线，最终替换主线当前的 Vanilla JS 前端层。

---

## 一、与主线的关系

```
主线 (ling-server-dashboard/)          dame (前端实验)
══════════════════════════            ══════════════════
FastAPI 后端          ✅               无后端（纯静态）
WebSocket 推送        ✅               无（模拟数据）
PTY 终端              ✅               无（打字演示）
Docker SDK            ✅               无（随机数据）
SQLite + 迁移         ✅               无（localStorage）
JWT 认证              ✅               无（仅登录页 UI）
psutil 指标           ✅               无（模拟引擎）
────────────────────────              ──────────────────
Vanilla JS SPA        ✅  ← 替换 ←    ✅ 重写版
ES Modules 分文件     ✅  ← 合并 ←    ✅ 单文件内联
Chart.js CDN          ✅  ← 保留 ←    ✅ 相同用法
xterm.js CDN          ✅  ← 保留 ←    ✅ 相同用法
CSS 分文件 (7个)      ✅  ← 替换 ←    ✅ 内联 <style>
```

**目标：dame 的前端代码 → 反哺主线 → 主线获得 dame 的视觉层，保留真实后端能力。**

## 二、dame 相比主线的前端改进

| 维度 | 主线现状 | dame 改进 |
|------|---------|----------|
| **主题** | OKLCH 令牌 + 暗/亮双主题 | 同体系 + 主题切换扩散波纹 + 3 层交错过渡动画 |
| **背景** | 图片 + SVG + Canvas 飘带 + 网格 + 光晕 | 同结构 + 控制栏可开关图层 + 图片池轮换 |
| **动画** | 少量（restore-in, spin） | 28 个 keyframes，全控件交互反馈，prefers-reduced-motion |
| **登录页** | 无独立登录页（SPA 内切换） | 独立页面 + 粒子爆发 + 双层玻璃卡片 |
| **快捷入口** | 无 | 平铺/图标双视图 + localStorage CRUD |
| **系统服务** | systemctl 白名单（后端） | localStorage CRUD + 启停开关 + 状态脉冲 |
| **文件管理** | 后端白名单目录浏览 | localStorage 目录树 CRUD |
| **滚动条** | 浏览器默认 | 暗/亮主题适配（Webkit + Firefox） |
| **响应式** | 基础断点 | 5 级断点 + Tab 横向滚动 + scroll-snap |
| **蓝图** | 可复用 CSS/组件 | 全部内联，可提炼回 css/ 分文件 |

## 五、合并策略

### 推荐分阶段推进

```
第一阶段：样式层（低风险）
  css/tokens.css  ← 补充令牌
  css/effects.css ← 合并 28 个 keyframes
  css/components.css ← 补充弹窗、卡片、动画样式
  css/themes/dark.css, light.css ← 补充滚动条
  css/layout.css ← 补充响应式断点

第二阶段：外壳层
  index.html ← 替换背景结构（5 层动效 + 控制栏）
  js/screens/dashboard.js ← 注入主题波纹 + 时钟优化
  js/utils/ ← 加入 Toast、Confirm Modal

第三阶段：Tab 逐个替换
  js/tabs/overview.js ← 加入快捷入口 + 新 KPI 卡片
  js/tabs/services.js ← 保留后端 API，替换 UI（卡片式）
  js/tabs/files.js ← 保留后端 API，替换 UI（目录树）
  js/tabs/terminal.js ← 保留 PTY，替换配色/字体
  js/tabs/docker.js ← 保留后端 API，替换卡片样式
  js/tabs/processes.js ← 保留后端 API，替换表格样式
  js/tabs/logs.js ← 保留后端 API，替换 UI
  js/tabs/alerts.js ← 保留后端 API，替换 UI

第四阶段：登录页（可选）
  新增独立登录页或改造 SPA 内登录为卡片式
```

### 合并原则

1. **后端 API 保持不变** — dame 的模拟数据引擎全部丢弃，改用主线已有的 REST + WebSocket
2. **样式优先** — 先合 CSS（零风险），再合 JS（需适配 API）
3. **localStorage 保留** — 快捷入口等功能无后端依赖，直接保留 localStorage 方案
4. **不破坏现有功能** — 每个 Tab 独立，可逐个替换，出问题立即回退

## 七、技术决策记录

| 决策 | 原因 |
|------|------|
| dame 选单文件内联 | 快速实验，无构建依赖，改完即看 |
| 合并时回拆为分文件 | 主线 ES Modules + CSS 分文件更适合维护 |
| 不使用框架 | 与主线保持一致（Vanilla JS 约定） |
| OKLCH 色彩空间 | 比 RGB/HSL 更适合暗色主题，主线已采用 |
| Fira Code 等宽字体 | 数据密集型面板的最佳选择 |
| localStorage 模拟 | 纯前端无法访问真实服务器，用浏览器存储验证交互逻辑 |
| prefers-reduced-motion | 必须 — 主线也有此规则 |
