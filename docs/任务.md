## 一、后端补齐（先做，耗时约 30 分钟）

目标：让主线后端 API 完整支持 dame 的所有 UI 功能。

### 需要补充的接口

| 功能 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 文件保存 | PUT | `/api/files/save` | 接收 `{path, content}`，写入文件（需白名单校验） |
| 文件移动/重命名（可选） | PUT | `/api/files/rename` | 接收 `{old_path, new_path}`，移动文件 |
| 快捷入口 | 无 | - | 纯前端 localStorage，无需后端 |

### 实施步骤

1. 在主线项目根目录，执行：
   ```bash
   git checkout -b feat/merge-dame-ui
   ```

2. 让 Claude 在 `server/routers/files.py` 中添加：
   ```python
   @router.put("/save")
   async def save_file(
       req: FileSaveRequest,
       current_user: dict = Depends(get_current_user)
   ):
       # 1. 检查白名单 (_safe_path)
       # 2. 写入文件 (使用 'w' 模式，UTF-8)
       # 3. 返回成功
   ```
   同时添加 `FileSaveRequest` schema 到 `server/models/schemas.py`。

3. 添加对应的测试：`tests/test_files.py` 中增加 `test_save_file`。

4. 运行 `./test.sh` 确保全部通过。

## 二、前端分阶段合并（按顺序执行）

> **原则**：每次只改一个方面，改完立即测试，可独立回滚。

### 阶段 0：准备工作

- 备份当前主线的关键 JS 文件（或者直接 commit 一个 baseline）。
- 从 dame 项目中提取需要的内容：
  - dame 的 `仪表盘/index.html` → 复制到临时文件夹，作为样式和结构参考。
  - dame 的 CSS 部分（内联 `<style>`）拆解为独立文件备用。

---

### 阶段 1：CSS 样式层合并（纯 CSS，零风险）

**目标**：让主线拥有 dame 的所有视觉效果（动效、滚动条、响应式断点、背景图层控制栏外观）。

**操作步骤**（由 Claude 完成）：

1. 创建/更新以下 CSS 文件：
   - `css/tokens.css`：补充 dame 中的新 CSS 变量（如 `--ease-spring`、`--bg-grid-opacity` 等）。
   - `css/effects.css`（新建）：复制 dame 中的所有 `@keyframes` 和动效类（`.pulse`, `.ripple` 等）。
   - `css/components.css`：补充 dame 中的卡片、弹窗、按钮动画样式。
   - `css/themes/dark.css` 和 `light.css`：添加滚动条样式（`::-webkit-scrollbar` 等）。
   - `css/layout.css`：添加 5 级响应式断点媒体查询。

2. 修改 `index.html`：
   - 保留原来的 `<div id="app">` 结构。
   - 在 `<body>` 内部，**增加 dame 的背景层结构**（5 层：canvas 飘带、网格、光晕等），但暂时不添加控制栏的 JS（仅静态层）。
   - 确保背景层位于 `#app` 下方（z-index 较低）。

3. 验证：
   - 页面样式与 dame 基本一致（颜色、圆角、阴影、动画过渡）。
   - 所有 Tab 内容正常显示，无布局错位。

---

### 阶段 2：外壳层与通用组件（低风险）

**目标**：替换顶部导航、Tab 切换逻辑、主题切换波纹、Toast/Modal 组件。

**操作步骤**：

1. **重构 `js/screens/dashboard.js`**：
   - 保留 `TAB_DEFS` 数组（Tab 列表不变）。
   - 将 dame 的 Header HTML 结构（包含时钟、主题切换按钮、用户名）合并进来。
   - 增加主题切换时的波纹效果：监听主题按钮点击，动态添加 `.ripple` 类后移除。
   - 确保懒加载 `loadTab()` 和 `cleanup()` 机制不变。

2. **新建 `js/utils/toast.js`**（或直接替换原有的 `notify.js`）：
   - 实现 dame 的 4 级通知（info/success/error/warning），支持自动消失。
   - 导出 `toast(message, type)`。

3. **新建 `js/utils/modal.js`**：
   - 实现 Promise 式的确认对话框 `confirm(message)` 和 `alert(message)`。
   - 使用 dame 的弹窗样式。

4. **修改 `index.html`**：引入上述新 JS 模块（ES module）。

5. 验证：
   - 主题切换时整个页面有波纹扩散效果。
   - 调用 `toast.success('ok')` 出现通知。
   - 调用 `confirm('确定删除？')` 返回 Promise。

---

### 阶段 3：逐个 Tab 替换（中等风险，每次一个 Tab）

**替换顺序**（从简单到复杂）：

1. `alerts.js` → 2. `services.js` → 3. `processes.js` → 4. `logs.js` → 5. `docker.js` → 6. `files.js` → 7. `terminal.js` → 8. `overview.js`

**通用替换方法**（对每个 Tab）：

- 打开 `js/tabs/xxx.js`（主线现有文件）。
- 用 dame 对应模块的 **HTML 结构和样式** 替换原来 UI，**但保留数据获取和提交的 API 调用**。
- 删除所有对 `localStorage` 或模拟数据的引用，改用：
  - REST：`import { get, post, put, del } from '../api.js'`
  - WebSocket：使用主线的 `ws.js` 中的全局连接（`/ws/live` 已在 `overview` 使用）。
- 确保导出 `render(container)` 和 `cleanup()` 函数。

**具体注意事项**：

| Tab | 注意事项 |
|-----|----------|
| `alerts.js` | 调用 `/api/alerts/rules` 和 `/api/alerts/history`；告警推送通过 `ws.js` 监听 `alert` 事件。 |
| `services.js` | 调用 `/api/services` 获取状态；启停操作使用 `post(/api/services/{name}/{action})`，需显示 loading。 |
| `processes.js` | 调用 `/api/processes?sort=mem&page=1`；终止进程用 `del(/api/processes/{pid})`。 |
| `logs.js` | 调用 `/api/logs?level=error&regex=...`；前端过滤 UI 需对应转换参数。 |
| `docker.js` | 调用 `/api/docker/containers` 等；检查 Docker 不可用时显示友好错误（主线已有机制）。 |
| `files.js` | 调用 `/api/files/list?path=...`；文件保存用新的 `PUT /api/files/save`；上传用 `FormData`。 |
| `terminal.js` | **保留主线原有的 PTY + xterm.js 核心逻辑**，仅将 xterm 主题和右键菜单样式替换为 dame 的配色和字体。 |
| `overview.js` | 最为复杂：KPI 数据来自 `/ws/live`；图表使用 `charts.js` 和 `canvas-wave.js`；快捷入口（localStorage）直接移植 dame 的 CRUD 逻辑，不依赖后端。 |

**验证每个 Tab**：
- 切换到该 Tab，所有数据能正常加载。
- 执行主要操作（创建告警、停止服务、杀死进程、上传文件、打开终端等）正常。
- 切换离开时，该 Tab 的 `cleanup()` 能停止定时器/WebSocket 订阅。

### 阶段 4：登录页与收尾

**目标**：将登录页样式升级为 dame 的玻璃卡片 + 粒子背景。

**操作**：
- 修改 `js/screens/login.js`，替换内部表单的 HTML/CSS 为 dame 登录页的样式。
- 添加 Canvas 粒子背景（复制 dame 登录页中的 `<canvas>` 和粒子动画 JS）。
- 保留原有的登录认证逻辑（`auth.js`）。

**验证**：
- 未登录访问根路径，显示新登录页。
- 输入正确的 admin 密码，登录后进入仪表盘。
- 登出后回到登录页。
